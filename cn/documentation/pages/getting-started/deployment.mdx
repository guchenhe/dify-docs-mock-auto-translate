# 部署指南

了解如何在生产环境中部署 Dify。

## 部署选项

Dify 可以通过多种方式部署：

### 云部署

在流行的云平台上部署 Dify：

- **AWS**：使用带有 Docker 的 EC2 实例
- **Azure**：使用 Azure 容器实例部署
- **Google Cloud**：使用 GKE 进行 Kubernetes 部署

### 自托管部署

在您自己的基础设施上运行 Dify：

1. **Docker Compose**（推荐小团队使用）
   - 快速设置，配置最少
   - 适合开发和测试

2. **Kubernetes**（推荐生产环境使用）
   - 可扩展且具有弹性
   - 支持高可用性

## 环境配置

需要配置的关键环境变量：

```bash
# Database
DB_HOST=localhost
DB_PORT=5432
DB_USERNAME=postgres
DB_PASSWORD=your_password

# Redis
REDIS_HOST=localhost
REDIS_PORT=6379

# API Keys
OPENAI_API_KEY=your_api_key
```

## 监控

为您的 Dify 部署设置监控：

- 在 `/health` 端点进行健康检查
- 使用 Prometheus 收集指标
- 使用 ELK 堆栈进行日志聚合

## 安全最佳实践

- 在生产环境中使用 HTTPS
- 启用身份验证
- 定期更新依赖项
- 正确配置防火墙
