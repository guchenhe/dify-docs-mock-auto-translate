Dify 的真正价值在于您可以轻松构建、部署和扩展任何复杂的 AI 想法。它专为快速原型开发、流畅迭代和任何级别的可靠生产部署而构建。

让我们首先学习如何将可靠的大型语言模型集成到您的应用程序中。在这个全面的指南中，您将构建一个简单而强大的聊天机器人，它可以分类用户的问题，使用大型语言模型直接响应，并通过有趣的国家特定趣味知识来增强响应。

## 步骤 1：创建新工作流（2 分钟）

1. 前往 **工作室** > **工作流** > **从空白创建** > **编排** > **新对话流** > **创建**

## 步骤 2：添加工作流节点（6 分钟）

<Tip>
  当您想要引用任何变量时，请先键入 `{` 或 `/`，您可以看到工作流中可用的不同变量。
</Tip>

### 1. 大型语言模型节点和输出：理解并回答问题

<Info>
  `大型语言模型` 节点向语言模型发送提示词以根据用户输入生成响应。它抽象了 API 调用、速率限制和基础设施的复杂性，因此您可以专注于设计逻辑。
</Info>

<Steps>
  <Step title="创建大型语言模型节点">
    使用 `添加节点` 按钮创建一个大型语言模型节点，并将其连接到您的开始节点
  </Step>
  
  <Step title="配置模型">
    选择一个默认模型
  </Step>
  
  <Step title="设置系统提示词">
    将以下内容粘贴到系统提示词字段中：

    ```text
    The user will ask a question about a country. The question is {{sys.query}}  
    Tasks: 
    1. Identify the country mentioned. 
    2. Rephrase the question clearly. 
    3. Answer the question using general knowledge. 
    
    Respond in the following JSON format: 
    {   
      "country": "<country name>",
      "question": "<rephrased question>",
      "answer": "<direct answer to the question>" 
    }
    ```
  </Step>
  
  <Step title="启用结构化输出">
    **启用结构化输出** 让您能够轻松控制大型语言模型将返回什么，并确保一致的、机器可读的输出，用于下游精确数据提取或条件逻辑中的使用。
    
    - 切换输出变量结构化为开启 > `配置` 并点击 `从 JSON 导入`
    - 粘贴：
    
    ```json
    {   
      "country": "string",   
      "question": "string",   
      "answer": "string" 
    }
    ```
  </Step>
</Steps>

### 2. 代码块：获取趣味知识

<Info>
  `代码` 节点使用代码执行自定义逻辑。它让您可以在可视化工作流中恰好需要的地方注入代码，为您节省了搭建整个后端的工作。
</Info>

<Steps>
  <Step title="创建代码节点">
    使用 `添加节点` 按钮创建一个 `代码` 节点并连接到大型语言模型块
  </Step>
  
  <Step title="配置输入变量">
    将一个 `输入变量` 名称更改为 "country" 并将变量设置为 `structured_output` > `country`
  </Step>
  
  <Step title="添加 Python 代码">
    将以下代码粘贴到 `PYTHON3` 中：

    ```python
    def main(country: str) -> dict:     
      country_name = country.lower()     
      fun_facts = {
        "japan": "Japan has more than 5 million vending machines.",
        "italy": "Italy has more UNESCO World Heritage sites than any other country."     
      }     
      fun_fact = fun_facts.get(country_name, f"No fun fact available for {country.title()}.")
      return {"fun_fact": fun_fact}
    ```
  </Step>
  
  <Step title="重命名输出变量">
    将输出变量 `result` 更改为 `fun_fact` 以获得更好的标记变量
  </Step>
</Steps>

### 3. 回答节点：给用户的最终答案

<Info>
  `回答` 节点创建一个干净的最终输出以返回。
</Info>

<Steps>
  <Step title="创建回答节点">
    使用 `添加节点` 按钮创建一个 `回答` 节点
  </Step>
  
  <Step title="配置回答字段">
    将以下内容粘贴到回答字段中：

    ```text
    Q: {{ structured_output.question }} 

    A: {{ structured_output.answer }} 

    Fun Fact: {{ fun_fact }}
    ```
  </Step>
</Steps>
