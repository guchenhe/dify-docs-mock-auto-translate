Dify provides powerful APIs that allow you to integrate AI capabilities directly into your applications. This guide walks you through the essential steps to get started with API integration.

## Overview

The Dify API enables you to:
- Create and manage conversations
- Send messages to AI assistants
- Stream responses in real-time
- Access workflow outputs
- Manage user sessions

## Authentication

All API requests require authentication using an API key. You can obtain your API key from the Dify console.

```bash
curl -X POST 'https://api.dify.ai/v1/chat-messages' \
  -H 'Authorization: Bearer YOUR_API_KEY' \
  -H 'Content-Type: application/json'
```

## Basic Chat Integration

### Send a Message

```javascript
const response = await fetch('https://api.dify.ai/v1/chat-messages', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_KEY',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    inputs: {},
    query: "Hello, how can you help me today?",
    response_mode: "streaming",
    conversation_id: null,
    user: "user-123"
  })
});
```

### Handle Streaming Response

```javascript
const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  const lines = chunk.split('\n');
  
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = JSON.parse(line.slice(6));
      console.log(data.answer);
    }
  }
}
```

## Workflow Integration

For workflow-based applications, use the completion API:

```python
import requests

def run_workflow(inputs, user_id):
    response = requests.post(
        'https://api.dify.ai/v1/workflows/run',
        headers={
            'Authorization': 'Bearer YOUR_API_KEY',
            'Content-Type': 'application/json'
        },
        json={
            'inputs': inputs,
            'response_mode': 'blocking',
            'user': user_id
        }
    )
    return response.json()
```

## Error Handling

Always implement proper error handling for API calls:

```javascript
try {
  const response = await fetch(apiUrl, options);
  
  if (!response.ok) {
    throw new Error(`HTTP error! status: ${response.status}`);
  }
  
  const data = await response.json();
  return data;
} catch (error) {
  console.error('API call failed:', error);
  throw error;
}
```

## Rate Limits

Be aware of rate limits when making API calls:
- Free tier: 100 requests per hour
- Pro tier: 1000 requests per hour
- Enterprise: Custom limits

Implement exponential backoff for rate limit handling:

```javascript
async function apiCallWithRetry(url, options, maxRetries = 3) {
  for (let i = 0; i < maxRetries; i++) {
    try {
      const response = await fetch(url, options);
      if (response.status === 429) {
        await new Promise(resolve => setTimeout(resolve, Math.pow(2, i) * 1000));
        continue;
      }
      return response;
    } catch (error) {
      if (i === maxRetries - 1) throw error;
    }
  }
}
```

## Best Practices

1. **Secure API Keys**: Never expose API keys in client-side code
2. **Input Validation**: Always validate user inputs before sending to the API
3. **Error Handling**: Implement comprehensive error handling
4. **Caching**: Cache responses when appropriate to reduce API calls
5. **Monitoring**: Monitor API usage and performance
